# 📚 Niji Seikei Skill Book Vol.04：特殊ルール（条文の掟）

> [!TIP]
> **「条文」は神聖にして侵すべからず。** 独特の作法があるんや！

## 1. 「第◯条」の扱い 📜

規程や契約書に出てくる「第1条（目的）」みたいなやつ。
これ、見た目は見出しっぽいけど、Markdownでは**「本文（平文）」**として扱うのが鉄則や！

### 🚫 やらなくていいこと
*   見出し記号（#）はつけない！
*   太字（**）にもしない！
    *   （※指示がある場合を除くけど、基本は「そのまま」）

## 2. 空行（スペース）の特殊ルール 📏

ここが一番の「引っかけ問題」やから気をつけて！

### ⭕️ 正しい行間
条文と条文の間は、必ず**「1行空ける」**こと！

```text
第1条（目的）
本規程は、就業に関する事項を定める。
（ここを1行空ける！）
第2条（適用範囲）
社員全員に適用する。
```

### ❌ ダメな例
*   **詰まりすぎ**：読みづらい！
    ```text
    第1条（目的）
    本規程は〜。
    第2条（適用範囲）
    ```
*   **空けすぎ**：間延びする！
    ```text
    第1条（目的）
    本規程は〜。
    （空行）
    （空行）
    第2条（適用範囲）
    ```

## 3. 項（1, 2...）と号（(1), (2)...） 🔢

条文の中にも「親子関係」があるで。

*   **第1条**
    *   **1（第1項）**: 番号だけのやつ。
        *   **(1)（第1号）**: カッコ付きのやつ。

**【ポイント】**
*   「項」と「項」の間は**空行を入れない**！（詰めて書く）
*   「項」や「号」はインデント（Tab）を使って、階層を綺麗に見せること。
