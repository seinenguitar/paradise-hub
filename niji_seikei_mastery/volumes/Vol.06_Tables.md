# 📚 Niji Seikei Skill Book Vol.06：応用テクニック（魔のテーブル）

> [!TIP]
> **「崩れない表」を作る。** Markdownの表は繊細やから、優しく扱うんやで。

## 1. セル内改行の裏技 `<br>` ↩️

Markdownの表（テーブル）の中では、普通の「改行（Enter）」が効かへん！
改行すると表が壊れてまうねん。

### 🔧 解決策：HTMLタグ召喚
改行したい場所には、**`<br>`** って書くんや。

*   **PDF**:
    ```text
    | 項目 | 内容 |
    |-----|------|
    | 氏名 | 山田
    |      | 太郎 |
    ```
*   **Markdown**:
    ```markdown
    | 項目 | 内容 |
    | --- | --- |
    | 氏名 | 山田<br>太郎 |
    ```
    ※これなら1行に収まるし、プレビューではちゃんと改行されて見える！

## 2. セル結合（マージ）の処理 🧩

「縦に結合」とか「横に結合」されてるセル。
Markdownには「結合」機能がないんよ…。

### 🛡️ 対処法
1.  **「同じ内容」を書く**（基本）
    *   結合されてるセル全部に、同じ言葉を入れて埋める。
2.  **上だけ書いて、下は `同上`**（場合による）
    *   読みやすさ優先ならこっちもあり。迷ったら1の「全部埋める」が安全策！

## 3. 空行は絶対に入れない！ 🚫

表の途中に「空行」を入れると、そこで表が終わったとみなされて崩れる！
表のブロック（`|` で始まって `|` で終わる塊）は、絶対に連続させること。

*   **NG**:
    ```markdown
    | A | B |
    | - | - |
    | 1 | 2 |
    
    | 3 | 4 |  ←ここで別の表になっちゃう！
    ```

とにかく**「表の中ではEnterキー禁止」**！これテストに出るで！
